## 00 | auto 和 decltype

### decltype 推导规则

`decltype` 根据操作数的形式，有两套推导规则：

| 操作数形式 | 推导规则 | 示例 |
|---|---|---|
| 不加括号的变量名 / 类成员访问 | 返回声明类型 | `decltype(obj)` → `const Object` |
| 其他表达式（含加了括号的） | 左值 → `T&`，右值 → `T&&` 或 `T` | `decltype((obj))` → `const Object&` |

**核心口诀**：加括号就是表达式，按值类别推导（左值加 `&`）；不加括号看声明类型，原样照搬。

### 成员访问的陷阱

```cpp
struct Object { const int a; double b; };
const Object obj;
```

| 表达式 | 结果 | 原因 |
|---|---|---|
| `decltype(obj.a)` | `const int` | 不加括号，返回成员声明类型 |
| `decltype((obj.a))` | `const int&` | 加括号，左值表达式 |
| `decltype(obj.b)` | `double` | 不加括号，返回声明类型，**忽略 obj 的 const** |
| `decltype((obj.b))` | `const double&` | 加括号，obj 的 const 生效 |

**关键陷阱**：`decltype(obj.b)` 返回 `double` 而非 `const double`，因为不加括号时只看成员本身的声明类型，忽略外层对象的 const 限定。

### 表达式值类别（Value Categories）与 decltype 的联动

C++11 将表达式分为 3 种基本类别：

| 类别 | 通俗解释 | 判断方法 | 例子 |
|---|---|---|---|
| **lvalue**（左值） | 有名字、有地址 | 能取 `&` | `obj`, `*ptr`, `arr[0]`, `str` |
| **prvalue**（纯右值） | 临时对象、字面量 | 不能取 `&` | `42`, `true`, `Object()`, `a+b` |
| **xvalue**（将亡值） | 即将被移动的濒死左值 | `std::move` 的产物 | `std::move(obj)` |

**`(obj.b)` 为什么是左值？** `obj.b` 是具名成员变量，有地址，`&(obj.b)` 合法。括号只是语法分组，不改变值类别。

**与 decltype 的联动规则**：

| 表达式值类别 | decltype 推导 | 示例 |
|---|---|---|
| lvalue（左值） | `T&` | `decltype((obj.b))` → `const double&` |
| xvalue（将亡值） | `T&&` | `decltype(std::move(x))` → `int&&` |
| prvalue（纯右值） | `T` | `decltype(42)` → `int` |

**推导 `decltype((obj.b))` = `const double&` 的完整过程**：① `(obj.b)` 加括号走规则 2 → ② 表达式是左值，加 `&` → ③ `obj` 是 `const`，表达式类型是 `const double` → 最终 `const double&`。

## 01 | = default 和 = delete

### 编译器自动生成特殊成员函数的规则

C++ 有 6 个特殊成员函数：默认构造、析构、拷贝构造、拷贝赋值、移动构造、移动赋值。

**编译器"罢工"规则**：

> 用户写了**任何构造函数** → 默认构造函数不再自动生成；
> 用户写了**拷贝/析构** → 移动构造/赋值不再自动生成。

### = default — 显式要求编译器生成

```cpp
struct B {
    B(int x) { }       // 定义了带参构造 → 默认构造消失
    B() = default;      // 强制编译器重新生成默认构造
};
```

### = delete — 显式禁止

```cpp
struct UniquePtr {
    void* dataPtr;
    UniquePtr() = default;
    UniquePtr(const UniquePtr&) = delete;            // 禁止拷贝
    UniquePtr& operator=(const UniquePtr&) = delete; // 禁止拷贝赋值
    UniquePtr(UniquePtr&&) = default;                // 允许移动
    UniquePtr& operator=(UniquePtr&&) = default;     // 允许移动赋值
};
```

**unique_ptr 语义速记**："拷删移默" — 拷贝 `= delete`，移动 `= default`。

### = delete 可用于任何函数

```cpp
void func(int x) { }
void func(float) = delete;    // 禁止 float 隐式转换
void func(bool) = delete;     // 禁止 bool
void* operator new(size_t) = delete; // 禁止堆分配
```

被 `= delete` 的函数**参与重载决议**，匹配到就报编译错误，从而阻止隐式类型转换。

### 记忆技巧

```
编译器罢工触发条件：
  你写了任何构造函数 → T() 消失，需要 = default 救
  你写了拷贝/析构   → T(&&) 消失，需要 = default 救

补救手段：
  = default → "请帮我生成"（恢复）
  = delete  → "禁止使用"（封杀）
```

## 02 | virtual、override、final

### 三个关键字的位置与职责

| 关键字 | 谁写 | 含义 |
|---|---|---|
| `virtual` | 基类 | "这个函数可以被子类重写" |
| `override` | 派生类 | "我重写了基类虚函数，请编译器帮我确认" |
| `final` | 类 或 虚函数 | "到此为止" — 类不能被继承 / 函数不能被继续重写 |

### final 的两种用法

```cpp
// 用法 1：修饰类 — 禁止继承
struct A final { };

// 用法 2：修饰虚函数 — 禁止继续重写
struct B {
    virtual void func() final { }
};
```

### 纯虚函数（= 0）

```cpp
struct AudioPlayer {
    virtual void play() final { init(); play_audio(); }
    virtual void init() = 0;      // 纯虚函数：子类必须实现
    virtual void play_audio() = 0; // 纯虚函数：子类必须实现
};
```

- 包含纯虚函数的类是**抽象类**，不能实例化，只能定义指针/引用
- 派生类**必须**实现所有纯虚函数，否则自己也是抽象类
- 纯虚函数可以有默认实现，但派生类仍须显式重写

### 模板方法模式（Template Method Pattern）

```
基类定义框架（final）：
  play()  ← 不可重写，定义调用顺序
    ├── init()  ← 纯虚，子类各自实现
    └── play_audio() ← 纯虚，子类各自实现

框架固定，细节可变。
```

### `. ` 调用 vs `->` 调用

- **`. ` 调用**：看对象的静态类型，无多态，直接调用该类自己的函数
- **`->` 调用**：遇 `virtual` 才动态绑定（查虚函数表），否则看指针的静态类型

```cpp
A* a = &b;
a->func1();  // virtual → 动态绑定 → B::func1()
a->func2();  // 非virtual → 静态绑定 → A::func2()
```

## 03 | 后置返回类型（trailing return type）

```cpp
// 传统写法
int add(double a, int b);

// 后置返回类型（C++11）
auto add(double a, int b) -> int;

// 配合 decltype 推导模板返回类型
template<typename T1, typename T2>
auto add(const T1& a, const T2& b) -> decltype(a + b);
```

后置返回类型的主要优势：`decltype` 可以访问参数名（参数在 `->` 之前已声明）。

## 04 | 右值引用与临时对象生命周期

### 左值 vs 右值

| 值类别 | 例子 | 说明 |
|---|---|---|
| 左值 (lvalue) | `obj`, `*ptr` | 有名字、可取地址 |
| 右值 (rvalue) | `Object()`, `std::move(obj)` | 临时对象、将亡值 |

**有名字的变量 = 左值，`std::move()` 把左值转成右值。**

### 临时对象生命周期延长

| 引用类型 | 能否绑定右值 | 能否延长生命周期 | 能否修改 |
|---|---|---|---|
| `T&` | ❌ | — | — |
| `const T&` | ✅ | ✅ | ❌ 不能修改 |
| `T&&` | ✅ | ✅ | ✅ 可以修改 |

```cpp
Object&& objRef = Object();  // 右值引用：延长生命周期 + 可修改
objRef.data = 1;             // ✅ 可以修改（const T& 则不行）
```

## 05 | 移动语义

### 移动构造函数

```cpp
Buffer(Buffer&& other) : data { other.data } {
    other.data = nullptr;  // ① 窃取资源 ② 置空源对象
}
```

### 移动赋值运算符

```cpp
Buffer& operator=(Buffer&& other) {
    if (this != &other) {
        delete[] data;          // ① 释放旧资源
        data = other.data;      // ② 转移资源
        other.data = nullptr;   // ③ 置空源对象
    }
    return *this;
}
```

### std::move 的作用

`std::move()` 本身不移动任何东西，只是把左值**强制转成右值引用**，从而触发移动构造函数/移动赋值。真正做移动的是移动构造函数/移动赋值运算符。

### 何时自动移动，何时需要显式 std::move

| 场景 | 自动移动？ |
|---|---|
| `return` 局部变量 | ✅ 自动（或 NRVO 消除） |
| 接收临时对象 | ✅ 自动 |
| 对有名变量操作 | ❌ 需要 `std::move()` |

### 所有权转移的全链路

```
内存分配一次 → 通过 std::move 在多个对象间传递 → 释放一次

buff1 ──move──→ buff2 ──move──→ buff3 ──move──→ buff4
                (buff1.data = nullptr)  (buff2.data = nullptr)  ...
```

**移动的是资源（指针），不是对象本身；对象地址不同，但内部 data 指向同一块内存。**

## 06 | 限定作用域枚举（enum class）

### 传统 enum 的两大问题

1. **命名冲突**：枚举值直接暴露在外部作用域，两个 enum 不能有同名值
2. **隐式整数转换**：不同 enum 之间可以比较，编译器不报错（逻辑错误静默通过）

```cpp
enum Color { RED, ORANGE };
enum Fruit { Apple, ORANGE }; // ❌ ORANGE 重复定义

Color c = RED;
if (c == Apple) { }  // ✅ 编译通过，但逻辑荒谬
```

### enum class 的修复

```cpp
enum class Color { RED, GREEN, BLUE, ORANGE };
enum class Fruit { Apple, Banana, ORANGE };

Color c = Color::RED;               // 必须带作用域
if (c == Color::RED) { }            // ✅ 同类型比较
// if (c == Fruit::Apple) { }       // ❌ 编译错误！跨类型比较
```

### 对比总结

| | `enum` | `enum class` |
|---|---|---|
| 命名空间 | 无 | 有（`Color::RED`） |
| 隐式转 int | 自动 | 禁止 |
| 跨类型比较 | 静默通过 | 编译错误 |
| 底层类型 | 实现定义 | 默认 `int`，可指定 `enum class X : uint8_t` |

## 07 | constexpr 和字面值类型

### constexpr 核心含义
- 用于声明编译期可求值的函数或变量，将计算从运行时提前到编译时
- C++11 中 constexpr 函数限制：函数体只能有一条 return 语句；所有参数和局部变量必须是字面值类型

### 字面值类型 (Literal Type)
只有字面值类型才能用于 constexpr 上下文，值必须能在编译期确定：
- **合法类型**：算术类型 (`int`/`float`/`bool`)、指针、枚举、拥有 constexpr 构造函数的类
- **非法类型 (C++11)**：`std::string`、`std::vector` 等需要动态内存分配的类型

### 常见错误
```cpp
constexpr std::string to_string(const std::array<int, 3> &arr) { // ❌ 错误
    // std::string 在 C++11 中不是字面值类型，不能在 constexpr 函数中创建
}
```

## 08 | List Initialization 列表初始化

### 核心语法与优势
- 统一使用花括号 `{}` 初始化所有类型，避免多种初始化语法的混淆
- **核心安全特性：禁止窄化转换**，可能导致信息丢失的隐式转换会直接报编译错误
  - `int x{3.14};` ❌ 编译错误，浮点数转整数丢失精度
  - `char c{1024};` ❌ 编译错误，值超出 char 范围

### Most Vexing Parse（最令人烦恼的解析）
```cpp
Object obj1();   // ❌ 这不是创建对象！编译器将其解析为函数声明
Object obj1;     // ✅ 正确：传统方式创建对象，调用默认构造
Object obj1{};   // ✅ 推荐：C++11 列表初始化，绝对不会被解析为函数
```

## 09 | std::initializer_list

### 作用
轻量级代理，使自定义类支持列表初始化。当用 `{a,b,c}` 初始化对象时，编译器将列表打包为 `std::initializer_list` 传递给对应的构造函数。

### 常用操作
```cpp
MyVector(std::initializer_list<int> list) : mSize(list.size()) {
    // 遍历所有元素
    for (int elem : list) { ... }
    
    // 访问指定位置元素（必须先检查 size！）
    if (list.size() >= 1) {
        int first = *list.begin();           // 第一个元素
    }
    if (list.size() >= 2) {
        int second = *(list.begin() + 1);    // 第二个元素
    }
}
```

### 构造函数优先级
列表初始化 `{}` 会**优先匹配** `std::initializer_list` 构造函数，即使有其他重载也优先选择它：
```cpp
std::vector<int> v1(10);  // 常规构造：创建含 10 个元素的 vector
std::vector<int> v2{10};  // 优先匹配 initializer_list：创建含 1 个元素，值为 10
```

## 10 | Delegating Constructors 委托构造函数

### 解决的问题
消除多个构造函数间的代码重复，将所有核心初始化逻辑集中到一个"目标构造函数"，其他构造函数委托给它。

### 正确语法
必须在**成员初始化列表**中调用同类的另一个构造函数，不能在函数体内调用：
```cpp
// 目标构造函数：包含所有核心逻辑
Account(std::string id_, std::string name_, int coin_)
    : id(std::move(id_)), name(std::move(name_)), obj(std::move(name_)) { }

// 委托构造函数：在初始化列表中委托
Account() : Account("default_id", "default_name", 0) { } // ✅ 正确
```

### 核心规则
1.  委托构造函数不能在自己的初始化列表中再初始化任何成员，只能委托
2.  执行顺序：先执行目标构造函数的所有代码，再执行委托构造函数的函数体

### 常见陷阱（导致断言失败）
```cpp
Account() {
    // ❌ 错误：在函数体中调用构造函数会创建临时对象
    // 导致 Object 的构造函数被调用两次，construction_counter 变成 2
    obj = Object("default_name");
}
```

### 另一个致命陷阱：循环委托
```cpp
Object() : Object(0, 0) {}
Object(int x, int y) : Object() {} // 💀 互相委托，无限递归栈溢出
```

### 最佳实践
使用一个**主构造函数 (Master Constructor)** 集中所有逻辑，其他构造函数都委托给它，形成有向无环的委托链。

## 11 | 继承构造函数（Inherited Constructors）

### `using T::T` — 一行导入所有基类构造函数

```cpp
class ObjectB : public ObjectBase {
public:
    using ObjectBase::ObjectBase;  // 把基类所有构造函数"导入"到派生类
};
```

等价于编译器自动为每个基类构造函数生成转发：

```cpp
ObjectB(int x)    : ObjectBase(x) { }
ObjectB(double x) : ObjectBase(x) { }
ObjectB(std::initializer_list<double> list) : ObjectBase(list) { }
```

**适用场景**：派生类构造函数只需纯转发，无额外逻辑。如果需要额外操作（日志、校验等），仍需手动 `Derived(...) : Base(...) { ... }`。

### Mixin 模式（模板继承模板参数）

```cpp
template <typename T>
class NoCopy : public T { ... };  // 编译期装饰器
```

继承 `T` 的所有接口，同时叠加拷贝控制语义。`NoCopy<Point>` 就是一个"被禁用了拷贝的 Point"。

### 删除函数的重载决议回退机制

```cpp
p = std::move(other);  // other 是右值引用

// 编译器查找 operator=：
// 第1步：operator=(NoCopy&&)       → 被 = delete 了 → 跳过
// 第2步：operator=(const NoCopy&)  → 右值可绑定到 const 左值引用 → 选中！
```

**关键规则**：`= delete` 的函数存在声明且参与重载决议，但匹配到被 delete 的函数后编译器会继续寻找其他候选。这实现了"禁止移动，回退到拷贝"的语义。

## 12 | nullptr

### `std::nullptr_t` — nullptr 的专属类型

```cpp
decltype(nullptr)  // → std::nullptr_t（不是 void、不是 void*、不是 int）
```

| 特性 | 说明 |
|---|---|
| 隐式转换 | 可隐式转换为**任意指针类型**和**任意成员指针类型** |
| 不能转换 | 不能隐式转换为 `int`、`bool` 等整数类型 |

### nullptr vs NULL vs 0 的重载解析

```cpp
void foo(int);
void foo(void*);

foo(0);       // ❌ 调用 foo(int)，0 是整数
foo(NULL);    // ❌ 调用 foo(int)，NULL 通常是 0
foo(nullptr); // ✅ 调用 foo(void*)，nullptr 只匹配指针重载
```

**核心价值**：在有 `int` 和 `T*` 重载时，`nullptr` 是唯一能精确命中指针版本的方式。

### 模板类型推导中的 nullptr

```cpp
template<typename T>
T clone(const T& t) { return t; }

clone(0);       // T = int → 返回 int
clone(NULL);    // T = int → 返回 int
clone(nullptr); // T = std::nullptr_t → 返回 std::nullptr_t，可隐式转任意指针
```

## 14 | 类型别名（Type Alias）

### `using` 基本语法

```cpp
using Integer = int;     // 等价于 typedef int Integer;
using Real = double;     // 等价于 typedef double Real;
```

`using` 比 `typedef` 更直观："新名字 = 旧类型"，天然符合从左到右的阅读习惯。

### 别名模板（Alias Template）— `using` 的杀手锏

```cpp
// C++03 typedef：需要套一层 struct，非常繁琐
template<typename T>
struct MyMap { typedef std::map<T, std::string> type; };
MyMap<int>::type m;  // 使用时还要加 ::type

// C++11 using：一行搞定
template<typename T>
using MyMap = std::map<T, std::string>;
MyMap<int> m;  // 直接使用
```

### 类内部别名 — STL 标准做法

```cpp
struct MyStruct {
    using ValueType = int;   // 对外暴露类型信息
};

MyStruct::ValueType x = 42;  // 外部访问
```

STL 中无处不在：`std::vector<int>::value_type`、`std::string::size_type`。

### `std::add_pointer` — 给类型加一层指针

```cpp
std::add_pointer<int>::type   // → int*
std::add_pointer<int&>::type  // → int*（引用被移除再加指针）
std::add_pointer<int*>::type  // → int**
```

在模板中用它比直接写 `T*` 更安全，因为它会自动处理 `T` 是引用等边界情况。

### 封装 `_t` 风格别名

```cpp
template<typename T>
using my_add_pointer_t = typename std::add_pointer<T>::type;
// 使用：my_add_pointer_t<int>  → int*
```

## 15 | 可变参数模板（Variadic Templates）

### 核心机制：递归展开参数包

```cpp
template<typename T, typename... Args>   // ① 类型参数包
void print(T first, Args... args) {       // ② 函数参数包
    std::cout << first << " ";
    print(args...);                        // ③ 展开参数包，递归调用
}

void print() { }  // ④ 递归终止函数（基例），必须定义
```

### 展开过程可视化

```
print(1, "hello", 3.14)
  ├─ T=int, first=1, args={"hello", 3.14} → 输出 "1 " → print("hello", 3.14)
  │   ├─ T=const char*, first="hello", args={3.14} → 输出 "hello " → print(3.14)
  │   │   ├─ T=double, first=3.14, args={} → 输出 "3.14 " → print()
  │   │   │   └─ 终止 ✅
```

### 三个关键语法

| 语法 | 含义 |
|---|---|
| `typename... Args` | 声明**类型参数包**，可匹配 0~N 个类型 |
| `Args... args` | 声明**函数参数包**，可接收 0~N 个实参 |
| `args...` | **展开参数包**（`...` 在名字后面） |

> C++11 只能递归处理参数包，C++17 的折叠表达式（`(args + ...)`）可替代递归写法。

## 16 | 广义联合体（Generalized Unions）

### C++11 放宽的限制

| 限制 | C++03 | C++11 |
|---|---|---|
| 成员有默认初始化器 | ❌ | ✅ 最多一个成员可有 |
| 成员是非平凡类型（如 `std::string`） | ❌ | ✅（但 union 自身构造/析构变为 deleted） |

### Union 成员的赋值规则

| 成员类型 | 激活方式 | 销毁方式 |
|---|---|---|
| 平凡类型（`int`、`double`） | 直接赋值 `u.i = 42` | 不需要 |
| 非平凡类型（`string`、`vector`） | placement new `new (&u.s) std::string("x")` | 手动析构 `u.s.~basic_string()` |

### 手动析构的统一语法

```cpp
std::string s;
s.~basic_string();  // 等价于 s.~string()（basic_string<char> 是 string 的全名）

std::vector<int> v;
v.~vector();        // 语法：对象.~类型名()

int x = 42;
x.~int();           // 合法，但平凡析构什么也不做
```

所有类型的析构语法都一样：`对象.~类型名()`。平凡的调了无害，非平凡的必须调。

### 切换 union 成员的流程

```cpp
// 从 string 切换到 int
u.s.~basic_string();  // ① 先析构旧成员
u.i = 42;             // ② 直接赋值新成员（平凡类型）
```

### 标签联合体（Tagged Discriminated Union）

```cpp
enum class Tag { INTEGER, STRING };  // 标签枚举

struct Value {
    Tag tag;
    Data data;  // union { int i; std::string s; }

    Value(int val) : tag(Tag::INTEGER) {
        new (&data.i) int(val);
    }
    Value(const std::string& val) : tag(Tag::STRING) {
        new (&data.s) std::string(val);
    }
    ~Value() {
        if (tag == Tag::STRING) data.s.~basic_string();
        // int 是平凡类型，无需析构
    }
};
```

**核心模式**：Union 不记录活跃成员 → 外部 enum 标签跟踪 → 构造用 placement new → 析构按标签选择。C++17 的 `std::variant` 自动完成所有这些工作。

## 17 | POD 类型

### POD = Trivial + Standard Layout

```
POD = Trivial（平凡）AND Standard Layout（标准布局）
```

| 条件 | 含义 | 破坏条件 |
|---|---|---|
| **Trivial** | 构造/析构/拷贝/移动都是编译器默认生成，无额外逻辑 | 自定义构造/析构、虚函数、虚基类、非平凡成员 |
| **Standard Layout** | 内存布局与 C 兼容，成员访问控制统一 | 不同访问权限混用（public + private）、多继承、虚基类 |

### 判别示例

```cpp
struct A { int x; double y; };              // ✅ POD
struct B { A a; int z; };                   // ✅ POD（嵌套 POD 仍是 POD）
struct C { virtual void foo(); int x; };     // ❌ 非 POD（虚函数破坏 Trivial）
struct D { int x; private: int y; };         // ❌ 非 POD（不同访问权限破坏 Standard Layout）
```

### 类型萃取工具

```cpp
std::is_pod<T>::value              // 是否是 POD（C++20 已废弃）
std::is_trivial<T>::value          // 是否是平凡类型
std::is_standard_layout<T>::value  // 是否是标准布局
```

### POD 的核心优势：可 memcpy 安全拷贝

```cpp
struct Packet { uint32_t len; uint16_t type; uint16_t flags; };  // POD

Packet p1{42, 1, 0xFF};
Packet p2{};
std::memcpy(&p2, &p1, sizeof(Packet));  // ✅ 安全，POD 没有隐藏指针
```

非 POD 的 `memcpy` 会导致灾难：

```cpp
struct Bad { std::string name; virtual void foo(); };  // 非 POD
// memcpy 会复制虚表指针和内部堆指针 → 两个对象指向同一块堆内存 → double free 💥
```

### C/C++ 互操作：POD 作为桥梁

C 语言不懂 `std::string`、虚函数，只认识原始字节。C++ 与 C 通信时，必须把富类型拆解成 POD 结构体：

```cpp
// C++ 侧：富类型
struct Message { uint16_t type; std::string payload; };

// C 侧：POD 头部
struct MessageHeader { uint32_t len; uint16_t type; uint16_t flags; };  // POD

// 适配层：提取 POD 信息传给 C
extern "C" void c_send_header(const void* data, size_t size);

void send_message_to_c(const Message& msg) {
    MessageHeader header{static_cast<uint32_t>(msg.payload.size()), msg.type, 0};
    c_send_header(&header, sizeof(header));
}
```

### `static_cast` 从 `void*` 还原类型

```cpp
const void* data;                                          // 无类型指针
const auto* header = static_cast<const MessageHeader*>(data);  // 还原为有类型指针
g_last_header = *header;                                   // 按 POD 结构体解释内存
```

`static_cast` 不改变二进制位，只改变编译器看待这块内存的方式。前提是 `data` 确实指向合法且对齐的 `MessageHeader` 内存（POD 保证了这个前提）。

## 附录 | C++ 引用（T&）

### 引用 = 别名

```cpp
int x = 10;
int& ref = x;   // ref 是 x 的别名，指向同一块内存
ref = 20;       // 修改 ref 就是修改 x
```

### 引用的核心特性

| 特性 | 说明 |
|---|---|
| 必须初始化 | `int& r;` ❌ 编译错误 |
| 不可重新绑定 | 初始化后永久绑定到原对象 |
| 语法糖 | 使用时无需 `*` 解引用，直接当原变量用 |

### 指针 vs 引用

| | 指针 `T*` | 引用 `T&` |
|---|---|---|
| 可否为空 | 可为 `nullptr` | 不能为空 |
| 重新绑定 | `p = &b` 可以 | 不能 |
| 访问方式 | 需要 `*p` | 直接当变量用 |
| 适用场景 | 所有权转移、可选参数 | 函数参数传递、避免拷贝 |

### 三种引用形式

| 引用类型 | 绑定左值 | 绑定右值 | 可修改 | 典型用途 |
|---|---|---|---|---|
| `T&` | ✅ | ❌ | ✅ | 函数传出参数 |
| `const T&` | ✅ | ✅ | ❌ | 避免拷贝的只读参数 |
| `T&&` | ❌（需 `std::move`） | ✅ | ✅ | 移动语义 |

### 悬垂引用（Dangling Reference）

```cpp
int& bad() {
    int x = 10;
    return x;  // ⚠️ warning，不是 error！返回后 x 已销毁
}
```

编译器只给警告不报错，因为无法在编译期通用判断所有生命周期问题。实际后果是未定义行为（UB），可能崩溃、输出垃圾值、或侥幸正确。

## C++14 | 泛型 Lambda（Generic Lambda）

### 核心特性：`auto` 参数

C++14 允许在 lambda 参数列表中使用 `auto`，编译器隐式生成模板化的 `operator()`：

```cpp
// C++14 泛型 lambda
auto identity = [](auto x) { return x; };

// 编译器等价于：
struct __anonymous {
    template<typename T>
    auto operator()(T x) const { return x; }
};
```

### C++11 vs C++14

```cpp
// C++11：必须指定具体类型
auto add = [](int a, int b) { return a + b; };

// C++14：泛型 lambda，一个 lambda 适用所有类型
auto add = [](auto a, auto b) { return a + b; };
add(1, 2);       // int
add(1.5, 2.3);   // double
```

### 多个 `auto` 参数独立推导

```cpp
auto print_pair = [](auto a, auto b) { std::cout << a << ", " << b; };
print_pair(42, "hello");  // a=int, b=const char* — 类型可以不同
```

每个 `auto` 参数独立推导类型，不必相同，与普通模板函数行为一致。

### 底层本质

Lambda 是编译器自动生成的函数对象（重载 `operator()` 的 struct），泛型 lambda 就是这个 `operator()` 变成了模板函数：

```cpp
auto lambda = [](auto x) { return x * 2; };
lambda(5);     // 实例化 operator()<int>
lambda(3.14);  // 实例化 operator()<double>
```

### 前置知识链

```
auto 类型推导（C++11）→ 模板函数 → 函数对象（operator()）→ Lambda 表达式（C++11）→ 泛型 Lambda（C++14）
```

## 附录 | operator 关键字

### 运算符重载的两类定义方式

| 方式 | 语法 | 调用等价 |
|---|---|---|
| 成员函数 | `T operator+(const T& o) const { }` | `a + b` → `a.operator+(b)` |
| 全局函数 | `T operator+(const T& a, const T& b) { }` | `a + b` → `operator+(a, b)` |

`=`, `()`, `[]`, `->` 必须是成员函数，其他运算符两者皆可。

### operator= — 拷贝赋值 / 移动赋值

```cpp
// 拷贝赋值
Buffer& operator=(const Buffer& other) {
    if (this != &other) {
        delete[] data;
        data = new int[2];
        data[0] = other.data[0];
        data[1] = other.data[1];
    }
    return *this;  // 返回 *this 支持链式赋值 a = b = c
}

// 移动赋值
Buffer& operator=(Buffer&& other) {
    if (this != &other) {
        delete[] data;          // ① 释放旧资源
        data = other.data;      // ② 窃取资源
        other.data = nullptr;   // ③ 置空源对象
    }
    return *this;
}
```

### operator() — 函数调用运算符（Functor / Lambda 底层）

```cpp
// 普通函数对象
struct Greater {
    bool operator()(int a, int b) const { return a > b; }
};
Greater g;
g(3, 5);  // 调用 operator()

// Lambda 是 operator() 的语法糖
auto greater = [](auto a, auto b) { return a > b; };
// 编译器展开为带模板 operator() 的匿名 struct
```

### operator new — 内存分配

```cpp
// 禁止堆分配
struct NoHeap {
    void* operator new(size_t) = delete;
    void operator delete(void*) = delete;
};

// placement new（在已有内存上构造）
new (&buffer) std::string("hello");  // 不分配新内存，在 buffer 上构造
```

### operator<< — 流插入运算符

```cpp
struct Point {
    int x, y;
    friend std::ostream& operator<<(std::ostream& os, const Point& p) {
        return os << "(" << p.x << ", " << p.y << ")";
    }
};
// 必须是全局函数（左操作数是 std::ostream，不是 Point）
```

### 不可重载的运算符

`::`（作用域）、`.` 和 `.*`（成员访问）、`?:`（三元）不可重载。

### 运算符出现场景速查

| 运算符 | 出现场景 | 核心作用 |
|---|---|---|
| `operator=` | 拷贝/移动赋值 | 控制对象复制或转移资源 |
| `operator()` | Lambda、函数对象 | 让对象像函数一样被调用 |
| `operator new` | 自定义内存分配 | 接管内存分配/释放 |
| `operator new (placement)` | Union 中激活成员 | 在已有内存上构造对象 |
| `operator<<` | 流输出 | 自定义类型的打印格式 |

> **所有运算符重载本质上都是函数调用**：`a + b` 就是 `a.operator+(b)` 或 `operator+(a, b)`。
